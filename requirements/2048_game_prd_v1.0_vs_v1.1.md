## 🔄 核心变更

### 1. 游戏规则调整（核心变更）

#### 数字生成概率

**v1.0：**
- 初始方块：90% 概率为 2，10% 概率为 4
- 移动后生成：2 或 4（未明确概率）

**v1.1：**
- 初始方块：**70% 概率为 2，30% 概率为 4**
- 移动后生成：**70% 为 2，30% 为 4**（明确概率）

**变更原因：**
- 增加游戏难度（4 出现概率提升 3 倍）
- 统一初始和移动后的生成概率
- 明确规则描述，便于开发实现

**影响评估：**
- ⬆️ 游戏难度提升（数字 4 更频繁出现）
- ⬇️ 达到 2048 的时间可能缩短
- ✅ 规则更加明确，开发无歧义

### 新增结构：gameRules

**v1.0：** 无此字段

**v1.1：** 新增完整游戏规则结构
```json
{
  "gameRules": {
    "boardSize": "4x4",
    "initialTiles": 2,
    "tileSpawnProbability": {
      "2": 0.7,
      "4": 0.3
    },
    "movements": ["up", "down", "left", "right"],
    "mergeRule": "相同数字的相邻方块合并为两倍",
    "winCondition": "出现 2048 方块",
    "loseCondition": "棋盘填满且无法合并"
  }
}
```

**新增字段说明：**
- `boardSize`: 棋盘尺寸
- `initialTiles`: 初始方块数量
- `tileSpawnProbability`: **关键变更** - 数字生成概率
- `movements`: 支持的移动方向
- `mergeRule`: 合并规则描述
- `winCondition`: 胜利条件
- `loseCondition`: 失败条件

**价值：**
- ✅ 结构化游戏规则，便于程序读取
- ✅ 开发人员可直接使用 JSON 数据
- ✅ 未来支持配置化（如 5×5 棋盘变体）

### 格式优化

部分数组格式从单行改为多行（提升可读性）：
```diff
- "platforms": ["web", "ios", "android"]

+   "web",
```
